## Startup Commands

### Terminal 1: Start Backend Server
```bash
cd backend
npm install
npm run dev
```
- Backend will run on http://localhost:5000
- Check console for "Server running on port 5000"

### Terminal 2: Start Frontend Client
```bash
cd client
npm install
npm run dev
```
- Frontend will run on http://localhost:5173
- Check console for "Local: http://localhost:5173"

### Terminal 3: Deploy Smart Contract (One-time)
```bash
cd blockchain
npm install
npx hardhat run scripts/deploy.js --network sepolia
```
- Deploys CertificateRegistry contract to Sepolia
- Saves contract address to backend/config/contractData.json

### Terminal 4: Seed Admin User (One-time)
```bash
# With backend running, open browser to:
http://localhost:5000/api/seed-admin
```
OR
```bash
curl http://localhost:5000/api/seed-admin
```

## Troubleshooting

### Issue: localhost:5173 not opening
**Cause**: Frontend server not started
**Solution**: Run `cd client && npm run dev`

### Issue: Direct login page opening
**Cause**: No authentication token or backend not running
**Solutions**:
1. Ensure backend is running on port 5000
2. Check CLIENT_URL in .env is "http://localhost:5173"
3. Clear browser localStorage if needed
4. Login with admin credentials from .env

### Issue: Port 5000 already in use
**Solution**: Kill process using port 5000
```bash
# Windows
netstat -ano | findstr :5000
taskkill /PID <PID> /F

# Or change PORT in .env to 5001
```

### Issue: Smart contract deployment fails
**Cause**: Missing WALLET_PRIVATE_KEY or insufficient Sepolia ETH
**Solution**:
1. Add valid private key to .env
2. Get Sepolia ETH from https://sepoliafaucet.com/

## Access URLs
- Frontend: http://localhost:5173
- Backend API: http://localhost:5000
- Admin Login: Use credentials from .env ADMIN_EMAIL/ADMIN_PASSWORD

## Development Workflow
1. Start backend (Terminal 1)
2. Start frontend (Terminal 2)
3. Deploy contract (Terminal 3, one-time)
4. Seed admin (one-time)
5. Access http://localhost:5173 in browser
